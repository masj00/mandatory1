Server side rendering(SSR) / Routers

Learning goals:

Understands how code and its placement affects application memory and efficiency.
    • Understands the arguments for and against SSR vs. CSR when it comes to:
    • Load time: SSR sends ready HTML = faster initial load. CSR relies on JS = slower initial load.
    • Resources spent: SSR uses server CPU and CSR uses client CPU.
    • SEO: SSR is better i would say, since search engines can crawl HTML directly.
    • CORS: SSR avoids CORS since requests come from the server.
    Can argue for better ways to structure the code and how to avoid files that are too long.
